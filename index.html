The Safety Tax

How many hidden LLM calls sit between your prompt and your answer?

Pipeline:

False Positive Rate

47.8%

Llama Guard 3-1B flagged 107 of 224 legitimate software engineering outputs as unsafe. 95% of code generation steps rejected.

Pipeline Compound Rejection

96.4%

In a 3-step pipeline, the probability that at least one step is blocked. Expected 21.4 calls (614% overhead) with per-step retry.

After Category Override

1.3%

Excluding S6 (Specialized Advice) and S8 (Intellectual Property) drops the false positive rate from 47.8% to 1.3%. A configuration fix, not a compute fix.

Hidden Token Cost

2-4x

Each LLM guard re-ingests the productive output (~1,500 tokens) plus its own prompt. With 3-4 LLM guards per step, safety consumes 2-4x the productive token budget — invisible to the user.

Guard Confidence

0.94

Mean entropy (bits) across 224 outputs — 94% of the theoretical maximum for a coin flip. 0% of outputs reached ≥0.90 confidence. Every guard check is a coin-flip verdict at ~290ms of GPU inference.

CodeShield vs LLM Guard

0 vs 114

CodeShield (static analysis) found 0 CWEs in 144 code outputs. Llama Guard flagged 114 of the same outputs — all false positives. Static analysis is deterministic, GPU-free, and finds real vulnerabilities.

Production Multiplier

5-24x

Production security models require 15-365 guard checks vs the 3 measured in drift baseline. Each additional pipeline step multiplies the overhead linearly.

Retry Entropy

0.94

Guard decisions are near-random (0.94 bits entropy on a 1-bit decision). Retries face independent coin-flip verdicts — exponential backoff doesn't help when the guard itself is uncertain.
